Routing Groups

Upstream Providers

Each provider is an upstream service (e.g. DeepSeek / OpenAI / Anthropic). API key is shown once on creation; as admin you can view/copy it later.

Model Routes

An alias is the user-facing model name (e.g. deepseek-v4-pro), routed to this group's provider upstream model.

Tool Providers (built-in tool keys)

Provide keys for built-in tools (e.g. Tavily) exposed to LLMs in this group. When enabled, the router auto-injects tool schemas into OpenAI chat requests.

Self-issued API Keys (db_sk_)

Keys for clients (Claude Code, Cursor, OpenAI SDK). The model field uses the alias above. Plaintext is shown once; admin can re-retrieve it. Keys are bound to this group — a client's key can only route to this group's providers/routes/tools.
